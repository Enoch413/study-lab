# STUDY LAB 다음 단계 체크리스트

## 이 문서의 목적
이 문서는 비개발자인 기획자/운영자가 지금부터 어떤 순서로 개발을 진행시키고, 각 단계가 끝날 때 무엇을 직접 눌러보고 확인해야 하는지 정리한 문서다.

## 현재 결정된 큰 방향
- CODE LAB 로그인은 마지막에 붙인다.
- 그 전까지는 STUDY LAB 기능 자체를 먼저 완성한다.
- 학생은 방 목록 없이 바로 `입실하기`만 누른다.
- 입실 전 카메라 안내 문구가 먼저 나온다.
- 카메라를 실제로 켜야만 입실된다.
- 카메라를 거부하면 입실이 막힌다.
- 전체 공부방에서는 학생들이 서로 영상을 볼 수 있다.
- 강사/관리자는 전체 학생 영상을 CCTV처럼 볼 수 있다.
- 질문 요청을 강사가 수락하면 학생과 강사가 1:1 질문방으로 자동 이동한다.
- 질문이 끝나면 학생은 자동으로 전체방으로 복귀한다.
- 전체방에서는 학생 마이크가 항상 꺼져 있어야 한다.
- 입실 후 카메라가 꺼지면 즉시 경고를 띄우고, 10분 이상 꺼져 있으면 자동 퇴실시킨다.

---

## 전체 개발 순서

### 1단계. 로그인 없이 STUDY LAB 화면부터 띄우기
이 단계의 목표는 “로그인 없이도 학생 화면, 강사 화면이 일단 보이게 만드는 것”이다.

개발자가 해야 할 일:
- STUDY LAB 진입 화면 만들기
- 임시 진입 방식 만들기
- 예: `학생으로 보기`, `강사로 보기`, `관리자로 보기`
- 화면 이동이 되는지 확인

당신이 확인할 것:
1. STUDY LAB 첫 화면이 열리는가
2. 학생 모드와 강사 모드로 각각 들어갈 수 있는가
3. 화면이 PC와 모바일에서 너무 깨지지 않는가

이 단계가 끝났다고 볼 기준:
- 로그인 없이도 STUDY LAB 테스트 화면을 열 수 있다
- 학생/강사 화면 전환이 가능하다

아직 안 해도 되는 것:
- 실제 CODE LAB 로그인 연결
- 실제 학생 계정 연결

---

### 2단계. 학생 입실 기능 만들기
이 단계의 목표는 “학생이 카메라를 켜고 전체 공부방에 입실하는 것”이다.

개발자가 해야 할 일:
- 학생 대시보드 만들기
- `입실하기` 버튼 만들기
- 카메라 안내 문구 또는 팝업 만들기
- 카메라 권한 요청
- 카메라 영상이 켜졌을 때만 입실 완료 처리
- `퇴실하기` 버튼 만들기

당신이 확인할 것:
1. 학생이 `입실하기`를 누르면 안내 문구가 먼저 나오는가
2. 카메라를 허용하지 않으면 입실이 막히는가
3. 카메라를 켜면 입실이 완료되는가
4. 입실 후 학생 자신의 영상이 보이는가
5. `퇴실하기`를 누르면 정상적으로 나가지는가

이 단계가 끝났다고 볼 기준:
- 카메라를 켜야만 입실된다
- 카메라를 거부하면 입실되지 않는다
- 입실/퇴실이 정상적으로 된다

아직 안 해도 되는 것:
- 질문방 이동
- 강사의 전체 학생 영상 확인

---

### 3단계. 강사 현황판 만들기
이 단계의 목표는 “강사가 전체 학생을 CCTV처럼 보는 것”이다.

개발자가 해야 할 일:
- 강사 대시보드 만들기
- 전체 학생 목록 표시
- 학생별 영상 표시
- 학생별 오늘 공부 시간 표시
- 학생 검색 기능
- 질문 요청 상태 표시

당신이 확인할 것:
1. 강사 화면에서 학생들이 한눈에 보이는가
2. 학생 영상이 실제로 보이는가
3. 학생별 공부 시간이 보이는가
4. 학생 이름 검색이 되는가
5. 입실 중인 학생과 아닌 학생이 구분되는가

이 단계가 끝났다고 볼 기준:
- 강사가 전체 학생 현황을 바로 파악할 수 있다
- 영상 + 공부시간 + 질문상태가 함께 보인다

아직 안 해도 되는 것:
- 질문 수락 후 질문방 자동 이동

---

### 4단계. 질문 기능 만들기
이 단계의 목표는 “질문 요청부터 1:1 질문방 이동까지”다.

개발자가 해야 할 일:
- 학생 `질문 요청` 버튼 만들기
- 강사 질문 대기 목록 만들기
- 강사 `수락` 버튼 만들기
- 질문 수락 시 학생과 강사를 1:1 질문방으로 자동 이동
- 질문방에서도 영상 유지
- 질문방에서는 마이크 자동 ON

당신이 확인할 것:
1. 학생이 질문 요청을 할 수 있는가
2. 강사 화면에 질문 요청이 뜨는가
3. 강사가 수락하면 둘이 자동으로 질문방으로 이동하는가
4. 질문방에서도 영상이 그대로 유지되는가
5. 질문방에서는 마이크가 켜지는가

이 단계가 끝났다고 볼 기준:
- 질문 요청 -> 수락 -> 질문방 이동이 자동으로 된다

아직 안 해도 되는 것:
- 질문 종료 후 전체방 복귀

---

### 5단계. 질문 종료와 전체방 복귀 만들기
이 단계의 목표는 “질문이 끝나면 학생이 자동으로 전체방으로 돌아오는 것”이다.

개발자가 해야 할 일:
- 강사 `질문 종료` 버튼 만들기
- 질문 종료 시 학생 자동 복귀
- 복귀 후 전체방 마이크 강제 OFF
- 복귀 후 전체방 영상 다시 연결

당신이 확인할 것:
1. 강사가 질문 종료를 누를 수 있는가
2. 학생이 자동으로 전체방으로 돌아오는가
3. 돌아온 뒤 영상이 계속 보이는가
4. 돌아온 뒤 학생 마이크가 다시 꺼지는가

이 단계가 끝났다고 볼 기준:
- 질문 시작부터 종료 후 복귀까지 한 흐름이 완성된다

---

### 6단계. 운영 규칙 붙이기
이 단계의 목표는 “학원용 자습실답게 규칙을 강하게 적용하는 것”이다.

개발자가 해야 할 일:
- 카메라 OFF 즉시 경고 띄우기
- 카메라 OFF 시간이 10분 넘으면 자동 퇴실 처리
- 전체방 마이크 강제 OFF 유지
- 공부 시간 기록 안정화
- 인터넷 끊김/브라우저 종료 시 기록 이상 없게 처리

당신이 확인할 것:
1. 학생이 카메라를 끄면 바로 경고가 뜨는가
2. 다시 켜면 경고가 없어지는가
3. 10분 이상 꺼두면 자동 퇴실되는가
4. 전체방에서 마이크를 켤 수 없는가
5. 공부 시간이 이상하게 계산되지 않는가

이 단계가 끝났다고 볼 기준:
- 학원 운영 규칙이 제대로 적용된다

---

### 7단계. 마지막에 CODE LAB 로그인 붙이기
이 단계의 목표는 “임시 테스트 방식 대신 CODE LAB 실제 로그인과 연결하는 것”이다.

개발자가 해야 할 일:
- 임시 `학생/강사/관리자 모드` 제거
- CODE LAB 로그인 사용자 정보를 STUDY LAB에 연결
- 학생/강사/관리자 자동 구분
- 실제 계정 기준으로 권한 제한

당신이 확인할 것:
1. CODE LAB 로그인 후 STUDY LAB로 자연스럽게 들어가는가
2. 학생 계정은 학생 화면만 보는가
3. 강사 계정은 강사 화면을 보는가
4. 관리자 계정은 관리자 권한을 가지는가

이 단계가 끝났다고 볼 기준:
- 임시 테스트 모드 없이도 실제 사용자로 STUDY LAB을 사용할 수 있다

## 지금 바로 시작해야 하는 단계
지금은 **2단계 학생 입실 기능 만들기**부터 시작하는 게 가장 좋다.

이유:
- STUDY LAB의 핵심은 카메라를 켜고 입실하는 자습실이기 때문이다.
- 학생 입실이 먼저 완성돼야 강사 화면도 만들 수 있다.
- 질문방도 결국 학생이 먼저 입실해야 시작할 수 있다.

즉, 지금 제일 먼저 완성해야 하는 것은:
1. 학생 화면
2. 카메라 안내 문구
3. 카메라 필수 입실
4. 전체 공부방 입실
5. 퇴실
